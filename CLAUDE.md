# CLAUDE.md

이 파일은 Claude Code가 이 저장소에서 작업할 때 따라야 할 규칙을 정의합니다.

## 응답 규칙

- 사용자의 질문에 답변할 때, 답변 **마지막에 아래 3가지를 요약**한다. 각 항목 앞에 이모지를 붙인다.
  - **❓ 질문**: 사용자가 무엇을 물었는지 한 줄로
  - **✅ 결론**: 그에 대한 핵심 답변/결정 사항
  - **💡 권장(추천)**: 다음으로 권장하는 행동이나 선택지
- 요약은 작업 맥락 추적이 목적이므로, 작업이 많아져도 어떤 질문에 어떤 답을 했는지 한눈에 보이도록 간결하게 작성한다.
- 단순 확인성 대화(인사, 한 단어 응답 등)에는 생략 가능하다.

## 명령어 트리거

- **명령어는 대/소문자를 구분하지 않는다.** 예: `ship`, `Ship`, `SHIP`, `[Turbo]`, `/AFK` 모두 동일하게 인식한다.

- 사용자가 `ship`, `[ship]`, `/ship` 중 무엇으로 입력하든 **`.claude/commands/ship.md`의 절차를 그대로 실행**한다.
  - 요약: 미커밋 변경분 → 빌드 검증 → 커밋(+메시지) → 푸시 → (작업 브랜치면) main 머지 → main 푸시로 Vercel 자동 배포 트리거.
  - 빌드 실패·머지 충돌 시 중단하고 사용자에게 알린다. 강제 푸시 금지.
- 사용자가 `turbo`, `[turbo]`, `/turbo` 중 무엇으로 입력하든 **`.claude/commands/turbo.md`의 원칙으로 동작**한다.
  - 요약: 토큰 소모를 신경 쓰지 않고, 병렬화(동시 도구 호출·서브에이전트 fan-out)로 속도를 높이되 검증은 꼼꼼히. 비용보다 속도×정확도 우선.
- 사용자가 `afk`, `[afk]`, `/afk` 중 무엇으로 입력하든 **`.claude/commands/afk.md`의 원칙으로 동작**한다.
  - 요약: 자리 비움 모드. 되묻지 말고 가장 권장하는 방향으로 끝까지 진행. 단, 매우 위험하거나 비가역적인(배포·삭제·외부전송 등) 선택만 멈춰서 질문으로 남긴다.
