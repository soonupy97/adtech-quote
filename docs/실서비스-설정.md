# 실서비스 설정 가이드 (이메일 로그인 + 클라우드 저장)

백엔드 지식이 없어도 됩니다. 아래 순서대로 **클릭만** 따라 하면
"진짜 로그인"(서버 인증 · 비밀번호 암호화 · 계정별 데이터 격리)과
**비밀번호 재설정 메일**이 켜집니다.

소요 시간: 약 10분 · 비용: 무료(Supabase Free 플랜)

---

## 1. Supabase 프로젝트 만들기

1. https://supabase.com 접속 → **Start your project** → GitHub/구글 계정으로 로그인
2. **New project** 클릭
   - **Name**: `adtech-quote` (아무거나)
   - **Database Password**: 아무 비밀번호나 생성 후 **따로 메모** (나중에 거의 안 씀)
   - **Region**: `Northeast Asia (Seoul)` 권장
3. **Create new project** → 1~2분 기다리면 생성 완료

---

## 2. 데이터베이스 테이블 만들기 (스키마 실행)

1. 왼쪽 메뉴 **SQL Editor** → **New query**
2. 이 프로젝트의 `supabase/schema.sql` 파일 내용을 **전부 복사**해서 붙여넣기
3. 오른쪽 아래 **Run** 클릭 → "Success" 가 뜨면 완료

> 견적·거래처·품목·설정 등 모든 테이블과 보안 규칙(RLS)이 한 번에 생성됩니다.

---

## 3. 이메일 인증 설정 ★중요

이 앱은 **이메일을 로그인 수단**으로 쓰고, 비밀번호 분실 시
**재설정 메일**(Supabase 내장)로 복구합니다.

1. 왼쪽 메뉴 **Authentication** → **Sign In / Providers**(또는 **Providers**)
2. **Email** 항목 열기
3. **Confirm email**(이메일 확인) 옵션 — 가입 시 인증 메일을 보낼지 결정합니다.
   - **OFF (현재 운영 선택 · 가입 인증메일 없음)**: 가입 즉시 로그인됩니다. 인증 메일을 보내지 않습니다.
     앱은 가입 시 **도메인(MX) 검사 + 일회용 도메인 차단 + 이메일 중복 검사**로 가짜·중복을 거릅니다(`src/lib/email.ts`, `src/lib/auth.ts`).
     단, 도메인은 진짜인데 **앞부분이 가짜인 주소**(오타 등)는 막지 못하므로, 그런 계정은 비밀번호 재설정 메일을 받지 못할 수 있습니다.
   - **ON (선택 · 더 엄격)**: 가입하면 미인증 상태로만 생성되고 **받은 메일의 인증 링크를 눌러야** 로그인됩니다.
     없는/오타 주소는 활성화 불가 = 실제 수신 가능한 이메일만 가입(double opt-in). 단 **인증 메일이 실제로 도착해야 하므로
     4-2의 커스텀 SMTP 설정을 권장**합니다(기본 메일은 시간당 한도·스팸 분류로 미도착이 잦음).

> 인증 메일을 없애려면 이 토글을 **OFF** 로 두면 됩니다. 앱 코드는 ON/OFF 모두 자동 대응합니다(OFF면 가입 즉시 로그인).

> Free 플랜 기본 메일 발송은 **시간당 건수 제한**이 있어 테스트용입니다.
> 실운영에서 메일이 안정적으로 가게 하려면 4-2의 커스텀 SMTP를 설정하세요.

## 4. 메일 링크가 돌아올 주소 등록 ★재설정 메일 필수

비밀번호 재설정 링크를 누르면 앱의 `/reset-password` 페이지로 돌아와야 합니다.
이 주소를 Supabase에 허용 목록으로 등록해야 링크가 동작합니다.

### 4-1. Redirect URL 등록

1. **Authentication** → **URL Configuration**
2. **Site URL**: 앱 기본 주소 입력
   - 개발: `http://localhost:5173`
   - 운영: `https://내도메인` (배포 주소)
3. **Redirect URLs**에 아래를 **추가**(Add URL):
   - 개발: `http://localhost:5173/reset-password`
   - 운영: `https://내도메인/reset-password`
4. **Save**

> 이 등록을 빠뜨리면 재설정 메일의 링크를 눌러도 "redirect not allowed" 로 막힙니다.

### 4-2. (선택) 커스텀 SMTP — 실운영 권장

1. **Authentication** → **Emails**(또는 **SMTP Settings**)
2. 사용 중인 메일 발송 서비스(SendGrid, Mailgun, AWS SES, 회사 메일 등)의
   호스트·포트·계정·발신주소 입력 → 저장
3. **Email Templates** 에서 "Reset Password" 본문/발신자명을 한국어로 다듬어도 됩니다.

> SMTP를 설정하지 않으면 Supabase 기본 발송을 쓰며, 발송량 제한과 발신주소 고정 제약이 있습니다.

## 5. 연결 키 복사해서 .env 만들기

1. 왼쪽 메뉴 맨 아래 **Project Settings** → **API**
2. 다음 두 값을 복사:
   - **Project URL**
   - **anon public** key (Project API keys 섹션)
3. 이 프로젝트 루트의 `.env.example` 을 복사해 `.env` 파일을 만들고 값 입력:

```
VITE_SUPABASE_URL=https://xxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOi...(긴 문자열)
```

> `.env` 는 git 에 올라가지 않습니다(.gitignore 처리됨). 키 노출 걱정 없음.

---

## 6. 실행 & 계정 만들기

```
yarn install
yarn dev
```

1. 브라우저에서 `/login` 접속 → 화면 상단에 "로컬 목업 모드" 경고가 **사라졌으면** 연결 성공
2. **계정이 없으신가요? → 계정 만들기** 클릭
3. 이름 / 이메일 / 비밀번호 / 비밀번호 확인 입력 후 **계정 만들기**
   - Confirm email 이 ON 이면, 받은 인증 메일의 링크를 누른 뒤 로그인
4. 로그인 후 대시보드로 진입하면 끝

### 비밀번호 분실 시

1. 로그인 화면 → **비밀번호를 잊으셨나요?**
2. 가입한 이메일 입력 → **재설정 메일 보내기**
3. 메일의 링크 클릭 → `/reset-password` 페이지에서 새 비밀번호 입력 → 변경 완료

## 자주 묻는 것

**Q. 재설정 메일 링크를 눌렀더니 오류가 나요.**
→ 4-1의 **Redirect URLs**에 `/reset-password` 주소를 등록했는지 확인하세요.
배포 주소가 바뀌면 그 주소도 추가해야 합니다.

**Q. 재설정/인증 메일이 안 와요.**
→ 스팸함 확인. Free 기본 발송은 시간당 제한이 있으니 잠시 후 재시도하거나,
4-2의 커스텀 SMTP를 설정하세요.

**Q. 로컬 목업 모드(Supabase 미설정)에서도 비밀번호를 바꿀 수 있나요?**
→ 메일 발송이 안 되므로, 로그인 화면의 "비밀번호를 잊으셨나요?"에서
**이메일 + 이름**으로 본인 확인 후 즉시 재설정하도록 되어 있습니다(개발용).

**Q. 외부인이 마음대로 가입하지 못하게 하려면?**
→ 운영 단계에서 회원가입을 닫고 관리자만 계정을 발급하도록 바꿀 수 있습니다.
필요할 때 말씀해 주세요(가입 화면 숨김 + Supabase에서 직접 사용자 추가 방식).

**Q. 비용이 드나요?**
→ Supabase Free 플랜으로 충분합니다. 사용량이 커지면 그때 유료 전환을 검토하면 됩니다.
