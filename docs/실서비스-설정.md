# 실서비스 설정 가이드 (아이디 로그인 + 클라우드 저장)

백엔드 지식이 없어도 됩니다. 아래 순서대로 **클릭만** 따라 하면
"진짜 로그인"(서버 인증 · 비밀번호 암호화 · 계정별 데이터 격리)이 켜집니다.

소요 시간: 약 10분 · 비용: 무료(Supabase Free 플랜)

---

## 1. Supabase 프로젝트 만들기

1. https://supabase.com 접속 → **Start your project** → GitHub/구글 계정으로 로그인
2. **New project** 클릭
   - **Name**: `adtech-quote` (아무거나)
   - **Database Password**: 아무 비밀번호나 생성 후 **따로 메모** (나중에 거의 안 씀)
   - **Region**: `Northeast Asia (Seoul)` 권장
3. **Create new project** → 1~2분 기다리면 생성 완료

---

## 2. 데이터베이스 테이블 만들기 (스키마 실행)

1. 왼쪽 메뉴 **SQL Editor** → **New query**
2. 이 프로젝트의 `supabase/schema.sql` 파일 내용을 **전부 복사**해서 붙여넣기
3. 오른쪽 아래 **Run** 클릭 → "Success" 가 뜨면 완료

> 견적·거래처·품목·설정 등 모든 테이블과 보안 규칙(RLS)이 한 번에 생성됩니다.

---

## 3. 아이디 로그인 켜기 (이메일 인증 끄기) ★중요

이 앱은 아이디를 내부적으로 이메일 형태로 변환해 인증합니다.
실제 메일을 보내지 않으므로 **이메일 인증을 꺼야** 가입 즉시 로그인됩니다.

1. 왼쪽 메뉴 **Authentication** → **Sign In / Providers**(또는 **Providers**)
2. **Email** 항목 열기
3. **Confirm email**(이메일 확인) 옵션을 **OFF**로 변경 → 저장

> 이 설정을 안 하면 "계정 만들기" 후 로그인이 막힙니다.

---

## 4. 연결 키 복사해서 .env 만들기

1. 왼쪽 메뉴 맨 아래 **Project Settings** → **API**
2. 다음 두 값을 복사:
   - **Project URL**
   - **anon public** key (Project API keys 섹션)
3. 이 프로젝트 루트의 `.env.example` 을 복사해 `.env` 파일을 만들고 값 입력:

```
VITE_SUPABASE_URL=https://xxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOi...(긴 문자열)
```

> `.env` 는 git 에 올라가지 않습니다(.gitignore 처리됨). 키 노출 걱정 없음.

---

## 5. 실행 & 계정 만들기

```
yarn install
yarn dev
```

1. 브라우저에서 `/login` 접속 → 화면 상단에 "로컬 목업 모드" 경고가 **사라졌으면** 연결 성공
2. **계정이 없으신가요? → 계정 만들기** 클릭
3. 이름 / 아이디 / 비밀번호 입력 후 **계정 만들기**
   - 예: 아이디 `adtech`, 비밀번호 `adtd0010`
4. 바로 대시보드로 진입하면 끝 — 이후엔 같은 아이디/비밀번호로 로그인

---

## 자주 묻는 것

**Q. 가입 후 로그인이 안 돼요.**
→ 3단계(이메일 인증 끄기)를 안 했을 가능성이 큽니다. Confirm email 을 OFF 로 바꾸세요.
이미 만든 계정은 Authentication → Users 에서 해당 사용자의 ⋯ → **Confirm**(또는 삭제 후 재가입) 처리.

**Q. 외부인이 마음대로 가입하지 못하게 하려면?**
→ 운영 단계에서 회원가입을 닫고 관리자만 계정을 발급하도록 바꿀 수 있습니다.
필요할 때 말씀해 주세요(가입 화면 숨김 + Supabase에서 직접 사용자 추가 방식).

**Q. 비용이 드나요?**
→ Supabase Free 플랜으로 충분합니다. 사용량이 커지면 그때 유료 전환을 검토하면 됩니다.
